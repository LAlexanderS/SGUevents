## Руководство администратора мероприятий

Этот раздел описывает работу в **Кабинете администратора** для управления событиями, участниками и логистикой.

### 1. Кабинет администратора
Кабинет администратора — основное рабочее пространство. Здесь создаются мероприятия и осуществляется управление ими.

### 2. Типы мероприятий и их создание

#### Онлайн мероприятия
При создании онлайн-мероприятия открывается форма в Админпанели.
-   **Обязательные поля** подсвечены жирным шрифтом.
-   **Вложения**: Можно прикрепить файлы (презентации, материалы).
    !!! warning "Лимит размера"
        Размер каждого файла не должен превышать 10 Мб.
-   **Участники**: Можно добавлять пользователей по одному или массово.
-   **Галерея**: Добавление фотографий осуществляется через кнопку `+` в конце формы.

#### Доступные к посещению мероприятия
Для мероприятий с физическим присутствием или ограниченным доступом:
-   Есть возможность ограничить **общее количество мест**.
-   Система автоматически отслеживает количество **свободных мест**.

#### Оффлайн мероприятия
Особенность оффлайн-мероприятий — тесная интеграция с Telegram для коммуникации.

**Порядок настройки:**

1.  **Создайте группы в Telegram**:
    -   *Группа пользователей*: Для общения участников и анонсов.
    -   *Группа администраторов*: Для получения вопросов от участников (может быть общей для разных событий).
2.  **Узнайте ID групп**: Используйте бота `@username_to_id_bot`.
3.  **Заполните карточку мероприятия**:
    -   Внесите ID группы пользователей и ID группы администраторов.
    -   Добавьте ссылку на группу пользователей.
4.  **Добавьте бота**: Бот Платформы должен быть администратором в обеих группах.

!!! failure "Важно"
    Если поля ID не заполнены или бот не добавлен, связь с группами работать не будет.

### 3. Коммуникация

#### Связь с пользователями (Рассылка)
Администратор может написать пользователям через интерфейс кабинета:
1.  Нажмите на значок сообщения.
2.  Заполните форму отправки.
3.  Выберите получателей: **Все пользователи** мероприятия или **Конкретный пользователь**.
4.  Пользователь получит сообщение от бота (функция ответа на такие сообщения отключена).

#### Вопросы от пользователей
Пользователи могут задать вопрос через кнопку в карточке мероприятия.
-   Сообщение приходит в **Группу администраторов**.
-   Оно содержит кликабельную ссылку на профиль пользователя.
-   Ответ администратора происходит в личном чате с пользователем (переход по ссылке).

### 4. Логистика
Инструмент для управления перемещениями и размещением участников оффлайн-мероприятий.
-   Запись логистики создается администратором **на каждого участника** отдельно.
-   Внесенные данные можно редактировать по мере изменений.

## Техническое администрирование

Справочная информация для технических специалистов и DevOps.

### Зоны ответственности
- Управление справочниками (кафедры, площадки, типы событий).
- Модерация контента и заявок на организаторов.
- Настройка интеграций (SSO, Telegram, webhooks).
- Observability: метрики, логи, очереди задач.

### Панели управления

| Панель | Что содержит |
| :---: | :--- |
| **Django Admin** | Базовые модели, справочники, доступы |
| **«Мониторинг»** | Celery, очереди, отложенные задачи |
| **«Интеграции»** | Ключи API, проверка подключения, тестовые webhooks |

### Чек-лист перед релизом
- [ ] `python manage.py check --deploy`
- [ ] `mkdocs build` и публикация свежей документации.
- [ ] Обновление переменных окружения (`SITE_DOMAIN`, `TG_BOT_URL`, `SUPPORT_EMAIL`).

### Полезные команды
```bash
python manage.py clearsessions
python manage.py collectstatic --noinput
celery -A SGUevents worker -l info
mkdocs serve --dev-addr localhost:8088
```
